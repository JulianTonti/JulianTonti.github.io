[example] | [all] | [string] | [number] | [boolean] | [null] | [object] | [array] | [map] | [set] | [list] | [tuple] | [binary] | [format] | [enum] Available to all schema elements. ```javascript { "$schema" : "http://json-schema.org/draft/2019-09/schema#", "$id" : "#unique_id", "$ref" : "#id or #path/to/fragment", "$comment" : "some programmer's comment", "title" : "title of the element", "description" : "description of the element", "example" : ["example", "values", 1, 2.5, null, true], "default" : "fallback value if missing", "enum" : ["allowed", "values", 1, 2.5, null, true], "const" : "single value", "allOf" : [schema, schema, schema], // AND, all rules must be true "anyOf" : [schema, schema, schema], // OR, at least one rule must be true "oneOf" : [schema, schema, schema], // XOR, exactly one rule must be true "not" : schema, // the rule must not be true "if" : schema, "then" : schema, "else" : schema, } ``` * available to all schema nodes although not always meaningful * `$ref`is very useful as it allows code re-use and self referencing * `const` is essentially the same thing as `enum : [value]` * if a schema does not have a `type` value, `enum` can have mixed values * flexible conditional schemas can be built with `allOf|anyOf|oneOf|not|if|then|else` * unfortunately there is no `elif` but recursive `if|then` can be used instead
